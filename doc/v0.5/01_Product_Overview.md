# vibeCoding 工作流产品需求文档 (v0.4) - 产品架构与概述

## 1. 产品概述
### 1.1 背景
当前AI辅助编程已经成为趋势，针对碎片化的“问答式代码生成”，我们希望打造一款“vibeCoding”工作流驱动的**跨平台桌面端应用**。

### 1.2 目标与定位
本产品是一款支持 Windows 和 macOS 的桌面级应用。基于 **Tauri + Vue 3 + Naive UI** 构建。它具有极客风格的定制化现代 UI，底层核心调度引擎联动 **Claude**、**Codex** 和 **Gemini** 三大模型的 CLI 工具，实现由“需求 -> 分发 -> 开发 (逻辑+UI) -> 审查 -> 提交流程”的全自动化/半自动化闭环。

## 2. 核心架构与技术栈选型

### 2.1 技术栈选型
- **底层容器与系统调度**：Tauri (Rust)
  - *优势*：低内存占用，极小打包体积，原生的多进程通信能力(stdout/stdin 管道)，完美契合高频后台调度多个 CLI 的场景。
- **前端页面框架**：Vue 3 (Composition API)
- **UI 组件库**：Naive UI
  - *优势*：极客感强、高度定制化的主题配置能力，默认暗色模式出众，非常适合展示复杂的数据输出视窗、代码高亮和多维度面板的桌面端 IDE 类工具。

### 2.2 模型与角色分工
- **Claude (Tech Lead / 审查员 / Orchestrator)**：
  - **职责**：全局任务分解与派发、全局代码审查 (Code Review)、约束代码规范。它是工作流的“大脑中枢”。
- **Codex (Backend / Core Developer)**：
  - **职责**：接收后端的算法编写、核心业务逻辑实现、数据处理等纯代码层面的生产指令。
- **Gemini (Frontend UI/UX / BA)**：
  - **职责**：利用其强大的图文理解能力，负责前端页面的 UI/交互层面代码生成；同时负责读取并解析“带图”或“复杂排版”的需求文档/草图，将其转化为结构化研发任务。
