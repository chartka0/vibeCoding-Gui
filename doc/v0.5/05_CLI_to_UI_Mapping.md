# vibeCoding 工作流产品需求文档 (v0.5) - CCG CLI 与 Tauri 桌面的 UI 映射表

基于 CCG (Claude Code 编排 Codex + Gemini) 的原生 CLI 操作逻辑，为了在 Tauri + Naive UI 的桌面端实现“零门槛”的可视化操作，我们梳理了如下的命令与 UI 映射关系：

## 1. 核心开发工作流映射
这些是用户日常高频使用的命令，我们将它们封装为桌面端主界面的**“核心按钮组”**或**“流程式向导”**。

| 原生 CLI 命令 | 功能说明 | 桌面端 UI 映射 (Tauri + Naive UI) |
|--------------|--------|----------------------------------|
| `/ccg:workflow` | 6阶段完整工作流 | **首页醒目入口**：一个名为 [启动完整工作流] 的大按钮，点击后进入分步式的引导页 (Wizard)。 |
| `/ccg:plan` | 多模型协作规划 | **规划面板 (Planning Board)**：左侧为需求输入框，右侧实时生成实施计划清单 `.claude/plan/xxx.md` 并以 Kanban 或列表形式渲染。 |
| `/ccg:execute` | Claude 编排执行计划 | **计划评审卡底部的 [Approve & Build] 按钮**：点击后，界面切换至“构建监控模式”，展示全局进度环。 |
| `/ccg:codex-exec` | Codex 全权执行 | **快捷执行菜单**：在计划面板提供一个下拉选项 `[⚡ 极速构建 (Codex全权)]`，适合明确的纯逻辑后端任务。 |
| `/ccg:frontend` / `/ccg:backend` | 单端快速任务 | **快速操作台 (Quick Actions)**：提供两个悬浮按键，按键弹窗直接呼出文本输入框，输入一句需求直接派发给 Gemini (前端) 或 Codex (后端)。 |

## 2. 分析与质量保障映射
这些命令属于代码写完后的验证与调整阶段，我们将它们封装在**“诊断测试面板”**中。

| 原生 CLI 命令 | 功能说明 | 桌面端 UI 映射 (Tauri + Naive UI) |
|--------------|--------|----------------------------------|
| `/ccg:review` | 代码审查 (git diff) | **代码比对视窗 (Diff Viewer)**：类似 VSCode 的差异对比页面，左侧是旧代码，右侧是 Claude 审查后建议的新代码，配有 [接受] / [拒绝] 按钮。 |
| `/ccg:debug` / `/ccg:optimize` | 问题诊断与修复 | **一键诊断器**：当编辑器报错时，通过右键快捷菜单或报错气泡直接唤出 [🔍 智能诊断/优化]，结果输出在 Console 面板中。 |
| `/ccg:test` | 测试生成 | **测试覆盖率面板**：针对当前模块，提供 [生成单元测试] 按钮，并在旁边可视化展示覆盖率指标。 |

## 3. 规范驱动开发 (OPSX) 映射
将需求从模糊变为确定性的约束体系。这部分适合高级用户或大中型需求。

| 原生 CLI 命令 | 功能说明 | 桌面端 UI 映射 (Tauri + Naive UI) |
|--------------|--------|----------------------------------|
| `/ccg:spec-research` | 需求 → 约束集 | **约束定义画板 (Constraint Canvas)**：将抽象的需求转化为一条条具象的规则限制，在界面上以 Tag 或列表形式展示，允许用户拖拽增删。 |
| `/ccg:spec-plan` / `spec-impl` | 零决策计划执行 | **严格模式控制台**：在启动任务时提供一个 Switch 开关 `[开启 OPSX 强约束模式]`。开启后，必须所有约束校验飘绿才能通过。 |

## 4. Git 沉淀映射
取代枯燥的命令行。

| 原生 CLI 命令 | 功能说明 | 桌面端 UI 映射 (Tauri + Naive UI) |
|--------------|--------|----------------------------------|
| `/ccg:commit` | 智能提交 (conventional format) | **发布面板**：任务完成后自动弹出“准备提交”抽屉，智能填好遵循 Angular 规范的 Commit Message，允许用户微调后一键 `[🚀 Commit & Push]`。 |
| `/ccg:rollback` | 交互式回滚 | **时光机 (Timeline 分支树)**：提供可视化的历史节点，点击节点旁边的 [回滚] 按钮即可安全撤销。 |

---
**设计总结：**
我们不再要求用户去记忆繁杂的 `/ccg:*` 指令。所有的复杂编排，都被封装在 Naive UI 现代化的**按钮、表单、Diff 视窗和时间线组件**中。用户像在使用一款高阶的自动化办公软件一样，底层依然是 CCG 引擎在高速运转。
