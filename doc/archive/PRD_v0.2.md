# vibeCoding 工作流产品需求文档 (PRD v0.2)

## 1. 产品概述
### 1.1 背景
当前AI辅助编程已经成为趋势，针对碎片化的“问答式代码生成”，我们希望打造一款“vibeCoding”工作流驱动的**跨平台桌面端应用**。

### 1.2 目标与定位
本产品是一款支持 Windows 和 macOS 的桌面级应用。它通过结构化的需求文档驱动，并在核心调度引擎中联动 **Claude**、**Codex** 和 **Gemini** 三大模型的 CLI 工具，实现由“需求 -> 分发 -> 开发 (逻辑+UI) -> 审查 -> 提交流程”的全自动化/半自动化闭环。

## 2. 核心架构：多智能体(Multi-Agent)协作模型
本产品打破单一AI编码的限制，将不同的工作交由最擅长的模型来完成：

- **Claude (Tech Lead / 审查员 / Orchestrator)**：
  - **角色**：技术负责人与流程调度。
  - **职责**：负责任务分解与派发、全局代码审查(Code Review)、检查代码规范，并控制整体工作流的流转。
- **Codex (Backend / Core Developer)**：
  - **角色**：核心逻辑开发。
  - **职责**：负责复杂算法编写、核心业务逻辑实现、数据处理等纯代码层面的生产能力。
- **Gemini (Frontend UI/UX / BA)**：
  - **角色**：前端开发与业务分析。
  - **职责**：负责前端页面的 UI/交互层面代码生成、读取并解析图文并茂的需求文档/设计稿，将视觉和文档转化为结构化任务。

## 3. 核心用户故事
- **需求解析与派发**：作为产品规划人员，我提供需求文档后，系统底层的 Claude 会调用 Gemini 读取文档并解析UI需求，同时 Claude 自行拆解后端和逻辑任务。
- **多端协作编码**：系统自动调用 Codex 的 CLI 生成核心逻辑代码，调用 Gemini 的 CLI 生成前端组件代码。
- **自动化审查与测试**：代码生成后，Claude 对所有变更进行 Review，如果发现问题，打回给对应的模型进行修改。
- **交付与存档**：确认无误后，自动触发 Git 提交。

## 4. 标准工作流定义 (The vibeCoding Flow)

### Phase 1: 需求解析 (Idea -> Task)
- **动作**：用户输入需求或导入文档，**Gemini** 负责阅读理解文档并输出 UI/交互要点，**Claude** 结合这些要点拆解出开发任务列表。

### Phase 2: 并行开发 (Task -> Code)
- **动作**：
  - **Codex** CLI 接收核心代码与算法任务，专注输出业务逻辑。
  - **Gemini** CLI 接收前端绘制任务，专注输出结构化 HTML/CSS/UI框架代码。

### Phase 3: 审查与自愈 (Code -> Validated)
- **动作**：**Claude** 接管产出的所有代码，进行静态扫描和逻辑推演。如果报错或代码不规范，Claude 将生成修复建议并重新派发给 Codex 或 Gemini 进行 Self-healing。

### Phase 4: 交付存档 (Validated -> Repo)
- **动作**：所有测试与 Review 通过，应用自动执行 `git commit` 操作存档。
